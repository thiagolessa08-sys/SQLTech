REGRAS DE NEGÓCIO — DIM_BIORC_INSTITUCIONAL:
══════════════════════════════════════════
• CD_ORGAO é VARCHAR — SEMPRE use aspas simples: CD_ORGAO = '1' (NUNCA = 1)
• SECRETARIAS da prefeitura = poder executivo: CD_ORGAO = '1' E DS_UO <> DS_ORGAO
• CD_ORGAO = '2' → Câmara Municipal (LEGISLATIVO) — NÃO é secretaria
• CD_ORGAO negativo ('-1','-2','-3') → sem classificação — excluir
• Quando DS_UO = DS_ORGAO → linha genérica do órgão, NÃO uma secretaria
• SEMPRE que a pergunta for sobre "secretarias":
    AND i.CD_ORGAO = '1' AND i.DS_UO <> i.DS_ORGAO

══════════════════════════════════════════
REGRAS DE NEGÓCIO — RECEITA / ARRECADAÇÃO:
══════════════════════════════════════════
"Arrecadação", "receita", "o que a prefeitura arrecadou", "quanto entrou no caixa",
"quanto foi arrecadado" são SINÔNIMOS — todos vão para
FATO_BIORC_EXECUCAO_RECEITA + coluna VL_ARRECADACAO_RECEITA.

Toda pergunta de receita DEVE retornar TRÊS valores:
  1. Receita Bruta     → CD_TIPO_NATUREZA_RECEITA = 1
  2. Deduções          → CD_TIPO_NATUREZA_RECEITA = 2  (já são NEGATIVAS no banco)
  3. Receita Líquida   → Bruta + Deduções  (somar, não subtrair)

NUNCA use SUM(VL_ARRECADACAO_RECEITA) sem filtro — existe CD=-1 que infla o total.
SEMPRE filtre com IN (1, 2) ou via CASE WHEN.

Query modelo:
  SELECT
    SUM(CASE WHEN tn.CD_TIPO_NATUREZA_RECEITA = 1 THEN f.VL_ARRECADACAO_RECEITA ELSE 0 END) AS receita_bruta,
    SUM(CASE WHEN tn.CD_TIPO_NATUREZA_RECEITA = 2 THEN f.VL_ARRECADACAO_RECEITA ELSE 0 END) AS deducoes,
    SUM(CASE WHEN tn.CD_TIPO_NATUREZA_RECEITA IN (1,2) THEN f.VL_ARRECADACAO_RECEITA ELSE 0 END) AS receita_liquida
  FROM pref_aruja_sp.FATO_BIORC_EXECUCAO_RECEITA f
  JOIN pref_aruja_sp.DIM_BIORC_TIPO_NATUREZA_RECEITA tn ON f.SK_TIPO_NATUREZA_RECEITA = tn.SK_TIPO_NATUREZA_RECEITA
  JOIN pref_aruja_sp.DIM_BIORC_DATA_CALENDARIO d ON f.SK_DATA_CALENDARIO_ANO = d.SK_DATA_CALENDARIO
  WHERE d.NO_ANO = 2025

EXEMPLOS DE QUERY CORRETOS:
══════════════════════════════════════════
-- Despesas por secretaria (executivo, sem genéricos):
SELECT TOP 20 i.DS_UO AS secretaria,
  SUM(f.VL_SALDO_MES_PAGO) AS total_pago
FROM pref_aruja_sp.FATO_BIORC_MENSAL_INTERVENCAO_DOTACAO f
JOIN pref_aruja_sp.DIM_BIORC_INSTITUCIONAL i ON f.SK_INSTITUCIONAL = i.SK_INSTITUCIONAL
JOIN pref_aruja_sp.DIM_BIORC_DATA_CALENDARIO d ON f.SK_DATA_CALENDARIO_MES = d.SK_DATA_CALENDARIO
WHERE d.NO_ANO = 2025 AND i.CD_ORGAO = '1' AND i.DS_UO <> i.DS_ORGAO
GROUP BY i.DS_UO ORDER BY total_pago DESC

-- Despesas por função de governo:
SELECT TOP 10 func.DS_FUNCAO, SUM(f.VL_SALDO_MES_PAGO) AS total_pago
FROM pref_aruja_sp.FATO_BIORC_MENSAL_INTERVENCAO_DOTACAO f
JOIN pref_aruja_sp.BRIDGE_BIORC_FUNCIONAL_PROGRAMATICA b ON f.SK_FUNCIONAL_PROGRAMATICA = b.SK_FUNCIONAL_PROGRAMATICA
JOIN pref_aruja_sp.DIM_BIORC_FUNCAO func ON b.SK_FUNCAO = func.SK_FUNCAO
JOIN pref_aruja_sp.DIM_BIORC_DATA_CALENDARIO d ON f.SK_DATA_CALENDARIO_MES = d.SK_DATA_CALENDARIO
WHERE d.NO_ANO = 2025
GROUP BY func.DS_FUNCAO ORDER BY total_pago DESC

-- Inadimplência IPTU por bairro:
SELECT l.Bairro,
  SUM(l.Valor_Lancado) AS lancado,
  ISNULL(SUM(p.Valor_Pago),0) AS pago,
  SUM(l.Valor_Lancado) - ISNULL(SUM(p.Valor_Pago),0) AS inadimplente
FROM pref_aruja_sp.PREFEITURA_IPTU_LANCADO l
LEFT JOIN pref_aruja_sp.PREFEITURA_IPTU_PAGO p
  ON l.CPF_CNPJ = p.CPF_CNPJ AND l.Ano_Mes = p.Ano_Mes
WHERE LEFT(CAST(l.Ano_Mes AS VARCHAR(6)),4) = '2025'
GROUP BY l.Bairro ORDER BY inadimplente DESC
